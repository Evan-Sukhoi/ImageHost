P.1 背景

背景与痛点

前后逻辑：项目发起的前提，分析现状痛点以引出产品需求

品牌营销中心为多语言市场（德语、法语、荷兰语、英语）发布活动时需要同步产出多渠道视觉物料，现有流程高度依赖人工，集中在排版耗时、多语言适配、文案校对和多尺寸重复劳动四项核心痛点上。

排版设计耗时

P0

文案量大的线上线下物料排版耗时久，需反复调整布局。现有GPT-Image2仅提供参考效果，无法直接复用。

  • 人工排版时间长，迭代频繁
  • 参考级底图无法自动转化为成品

多语言文案版式适配

P1

逐语言替换文案易导致文字溢出、版式错位，需重新调整排版，缺乏文案替换+自适应排版自动化方案。

  • 多语言尺寸差异导致排版崩溃
  • 重调版式重复劳动严重

多语言文案校对

P2

图片文案校对依赖人工，现Gemini方案仅覆盖视频文案，图片侧无自动校对，稳定性不足。

  • 图片文案无自动校对
  • 人工校对成本高、易出错

多尺寸适配

P0

单张素材常需适配多达16个尺寸，图片与语种组合多时重复劳动剧增。

  • 多尺寸需求导致大量重复布局
  • 无批量自动适配工具
P.2 目标

产品目标

前后逻辑：基于痛点明确产品要达成的四个核心方向

构建AI驱动的视觉物料生成助手，实现多尺寸批量输出、多语种自动排版、多语言文案校对以及内置渠道规范与品牌资源，消除重复劳动并提升效率。

多尺寸批量输出

P0

基于渠道尺寸需求的设计底图，一键输出所有标准尺寸物料。

  • 自动适配不同渠道尺寸
  • 减少重复排版工作

多语种自动排版

P1

支持多语言文案替换与自适应排版，根据文案长度/语言特性自动调整版式。

  • 文案替换不引发版式错乱
  • 自适应调整字体与行距

多语言文案校对

P2

自动识别拼写、语法、语义错误，覆盖图片文案场景。

  • 覆盖图片文案校对缺口
  • 输出问题标注与建议

规范内置

P0

将渠道尺寸规范、品牌素材、多语言字体资源内置到系统中。

  • 设计师无需手动查阅规范
  • 统一管理字体与品牌资产

成功指标

前后逻辑：衡量产品目标实现程度的量化指标

以物料一次性通过率作为核心衡量，确保生成质量满足业务要求。

视觉物料一次性通过率> 90%
P.4 用户

目标用户

前后逻辑：定义系统的核心用户及其职责，指导功能设计

用户运营

活动策划执行，提供文案与目标尺寸，期望一键获得各语言物料。

  • 上传需求表格和素材
  • 选择渠道，预览并下载物料

品牌视觉

负责物料设计与质量把控，期望自动化处理重复排版，以便投入创意和审核。

  • 管理品牌资源与规范
  • 审核生成结果并微调

产品主要服务两类角色

  • 用户运营负责活动文案与诉求输入，需要快速获取多语种物料
  • 品牌视觉负责设计与审核，需要减少重复排版工作，聚焦创意。
P.5 功能

核心功能

前后逻辑：根据用户需求定义核心功能集，下层子模块按业务链路排列

系统提供五大能力

  • 多语种排版生成、多尺寸批量延展、规范与资源管理、批次管理及输出、多语言文案校对Agent。前四项优先P0/P1，文案校对为P2阶段功能。
F1 多语种物料排版生成

输入需求表格、底图PSD和元素切图，自动解析文案与坐标规则，渲染多语言合成稿，支持字体切换与自适应排版。

  • 输入：xlsx需求表（含各语言文案及尺寸）、底图PSD（不含文字）、元素切图PSD
  • 输出：按语言分组的合成终稿（DE/FR/NL/EN等）
  • 处理逻辑：解析xlsx→提取坐标规则→合成底图与元素→Pillow渲染文案→自适应字号/行距→溢出告警
F2 多尺寸批量延展

根据所选渠道自动输出所有标准尺寸物料，覆盖付费广告、社交媒体、论坛、领英、YouTube、PR及官网PC/移动端等渠道。

  • 覆盖7大渠道逾30种标准尺寸
  • 官网渠道分为PC（1920宽）和移动端（750宽）两套画布
F3 规范与资源管理

内置渠道尺寸规范与品牌资源库，统一管理多语言字体及品牌素材访问权限。

  • 渠道尺寸规范内置，免去设计师手动查阅
  • 品牌素材库权限管理（字体、Logo、色板等）
  • 多语言字体资源支持德/法/荷/英西欧字符集
F4 批次管理与输出

支持按批次组织物料，自动按渠道、语言、尺寸命名并打包下载整批次成品。

  • 按批次（如奥地利14批、德语14批）管理物料
  • 命名规范：[渠道]_[产品]_[批次]_[宽]x[高]px_[语言].jpg
  • 支持打包下载整批次物料
F5 多语言文案校对 Agent

自动对生成物料中的文案执行拼写、语法、语义校对，补充图片文案校对缺失，输出标注报告。

  • 覆盖图片文案场景，弥补Gemini仅视频校对的空缺
  • 检测项：拼写、语法、语义错误
  • 输出校对报告（问题位置+修改建议）
  • 支持语言：DE/FR/NL/EN，可扩展
  • 触发：文案替换后自动触发，也支持手动单张校对
P.6 流程

用户流程

前后逻辑：串联核心功能，描述从用户操作到交付的全过程

用户上传需求表格、底图PSD和元素切图，选择渠道后系统自动完成解析、合成、自适应排版、合规检测与文案校对，用户预览审核后可调整单张再重新生成，最终按规范命名打包下载。

用户输入

市场部/运营准备并上传所需文件，选择目标渠道。

  • 上传需求表格（xlsx）：各语言文案+目标尺寸+特殊要求
  • 上传底图 PSD（含各尺寸 Artboard，不含文字层）
  • 上传元素切图 PSD（产品图、图标等）
  • 选择目标渠道（可多选）
系统自动处理

后台依次完成解析、排版、合成、自适应渲染、合规检测与文案校对。

  • 解析xlsx提取各语言文案
  • 读取底图PSD Artboard获取元素坐标
  • 合成底图+元素切图
  • 按坐标规则渲染各语言文案（自适应排版）
  • 排版合规检测（溢出、安全区、分辨率）
  • 多语言文案校对自动触发
预览与审核

用户按语言/渠道/尺寸分组预览生成结果，查看校对报告，支持单张调整后重新生成。

  • 按语言、渠道、尺寸分组预览
  • 查看校对报告，标注文案问题
  • 支持单张调整后重新生成
批量导出

系统按统一命名规范打包，提供下载。

  • 按 [渠道]_[产品]_[批次]_[宽]x[高]px_[语言].jpg 命名
  • 打包下载整批次物料

P.8 技术

技术方向

前后逻辑：为满足功能和非功能需求选定的技术实现路线

采用psd-tools解析PSD图层坐标，Pillow进行底图与元素合成及文案渲染，利用历史排版规则实现坐标复用，通过项目内置Ageo TTF字体支持西欧字符集，输出为规范JPG。

输入解析

xlsx直接解析提取文案；PSD用psd-tools读取Artboard图层坐标和元素结构。

  • psd-tools处理图层结构
  • xlsx解析实现数据映射

图层合成与渲染

从底图PSD各Artboard导出背景，叠加元素切图，用Pillow渲染文字。

  • Pillow文字渲染
  • 合成底图与元素图层

排版规则

历史PSD提取各尺寸元素坐标规则，新任务直接套用，跨语言坐标高度一致。

  • 复用历史坐标规则
  • 跨语言高度一致性验证

字体管理

项目目录提供Ageo TTF文件，按字重加载，覆盖西欧字符集。

  • Ageo TTF统一字体
  • 支持DE/FR/NL/EN字符

输出格式

广告/社媒统一输出JPG，分辨率按渠道规范配置。

  • JPG输出
  • 渠道分辨率配置
